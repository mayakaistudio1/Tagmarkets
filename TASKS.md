## Telegram Admin Agent (Аналитик-бот)

### Подготовка
- ⬜ Создать Telegram-бота через @BotFather, сохранить токен как `TELEGRAM_BOT_TOKEN`
- ⬜ Узнать Telegram User ID админа (@userinfobot), сохранить как `TELEGRAM_ADMIN_ID`

### База данных
- ⬜ Добавить таблицу `admin_notes` (id, content, category, status, createdAt)
- ⬜ Методы хранилища: saveAdminNote(), getAdminNotes(), updateAdminNote()

### Бот-сервис (`server/telegram-bot.ts`)
- ⬜ Установить `grammy` (Telegram bot framework)
- ⬜ Языковая модель: GPT-4o (через Replit AI Integrations)
- ⬜ Системный промпт бота: полный контекст экосистемы JetUP, доступ к БД, роль аналитика
- ⬜ Ограничение доступа: только `TELEGRAM_ADMIN_ID`

### Команды бота
- ⬜ `/start` — приветствие, описание возможностей
- ⬜ `/today` — статистика за сегодня (сессии, сообщения, по языкам/типам)
- ⬜ `/stats` — общая статистика (всего сессий, активные языки и т.д.)
- ⬜ `/analyze text|video de|ru` — GPT-анализ (логика из admin panel)
- ⬜ `/note <текст>` — сохранить заметку/правку
- ⬜ `/notes` — показать все заметки
- ⬜ Свободный текст — GPT отвечает на вопросы о данных
- ⬜ Голосовые сообщения — транскрипция через Whisper, затем обработка как текст

### Админ-панель
- ⬜ API: GET/POST/PUT `/api/admin/notes`
- ⬜ Секция «Заметки» в Chat Logs (список заметок с датой, категорией, статусом)
- ⬜ Отметка заметок как «выполнено»

### Обновление промпта
- ⬜ Кнопка «Обновить промпт» — GPT собирает все активные заметки и формирует обновлённый промпт
- ⬜ Предпросмотр изменений перед применением
- ⬜ Подтверждение и сохранение

### Запуск
- ⬜ Интеграция в `server/index.ts` — polling рядом с Express-сервером
