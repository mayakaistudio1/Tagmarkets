## Telegram Admin Agent — AI-ассистент проекта

Личный AI-ассистент в Telegram. Знает весь проект JetUP, понимает неформальный стиль общения, имеет полную историю переписки (как ChatGPT), долгосрочную память и доступ к данным в реальном времени.

### Подготовка
- ⬜ Создать Telegram-бота через @BotFather → `TELEGRAM_BOT_TOKEN`
- ⬜ Узнать Telegram User ID админа через @userinfobot → `TELEGRAM_ADMIN_ID`

### База данных (3 новых таблицы)
- ⬜ `admin_notes` — заметки и правки для Марии (content, category, status, createdAt)
- ⬜ `agent_memory` — долгосрочная память (key-value, как Memory в ChatGPT Plus)
- ⬜ `bot_conversations` — полная история переписки с ботом (role, content, createdAt)
- ⬜ Методы хранилища для всех 3 таблиц

### Память (3 уровня, как в ChatGPT и лучше)
- ⬜ **История переписки** — каждое сообщение сохраняется, последние 50 подгружаются в контекст GPT (как в ChatGPT)
- ⬜ **Долгосрочная память** — ключевые факты, сохраняются навсегда, подгружаются в системный промпт (как Memory в ChatGPT Plus)
- ⬜ **Заметки** — конкретные правки и задачи, видны также в админке на сайте

### Бот-сервис (`server/telegram-bot.ts`)
- ⬜ Установить `grammy` (Telegram bot framework)
- ⬜ Языковая модель: GPT-4o (через Replit AI Integrations)
- ⬜ Безопасность: отвечает только `TELEGRAM_ADMIN_ID`, остальных игнорирует

### Системный промпт агента — знает:
- ⬜ Полная экосистема JetUP (продукты, услуги, партнёры, цены)
- ⬜ Архитектура проекта (React + Express + PostgreSQL + Drizzle)
- ⬜ Оба промпта Марии (Text Chat DE/RU/EN + Live Avatar)
- ⬜ Текущие задачи и бэклог проекта
- ⬜ Стиль общения админа: неформальный русский, голосовые, быстрые указания, на «ты»

### Команды бота
- ⬜ `/start` — приветствие, описание возможностей
- ⬜ `/today` — статистика за сегодня (сессии, сообщения, по языкам/типам)
- ⬜ `/stats` — общая статистика
- ⬜ `/analyze` — интерактивный GPT-анализ (text/video + язык)
- ⬜ `/note <текст>` — сохранить заметку (автокатегоризация: правка/идея/напоминание/задача)
- ⬜ `/notes` — показать активные заметки
- ⬜ `/done <id>` — отметить заметку выполненной
- ⬜ `/remember <ключ> <значение>` — сохранить в долгосрочную память
- ⬜ `/forget <ключ>` — удалить из памяти
- ⬜ `/memory` — показать всю память
- ⬜ `/clear` — очистить историю переписки (свежий старт, память и заметки остаются)

### Свободный текст и голос
- ⬜ Свободный текст — GPT отвечает на любые вопросы о проекте и данных
- ⬜ Голосовые сообщения — транскрипция через Whisper → обработка как текст
- ⬜ Может обсуждать идеи, помогать формулировать изменения, анализировать ответы Марии

## Идеи на будущее

### Push-to-Talk для Live Avatar
Вместо автоматической детекции речи — пользователь нажимает и держит кнопку, чтобы говорить. Отпустил — Мария обрабатывает и отвечает. Когда Мария говорит, кнопка автоматически блокируется. Никто никого не перебивает.

- ⬜ Добавить `interactivity_type: "PUSH_TO_TALK"` в payload сессии (`server/integrations/liveavatar.ts`)
- ⬜ Заменить кнопку микрофона на "нажми и держи" (press-and-hold)
- ⬜ Отправка событий `user.start_push_to_talk` / `user.stop_push_to_talk` через LiveKit
- ⬜ Автоблокировка кнопки когда Мария говорит (существующая логика)
- ⬜ Визуальные состояния: idle / говорите / Мария говорит
- ⬜ Поддержка тач-событий для мобильных и Telegram Mini App
- Документация: https://docs.liveavatar.com/docs/push-to-talk
